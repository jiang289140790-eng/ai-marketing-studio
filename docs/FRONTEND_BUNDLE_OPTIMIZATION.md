# 前端包体积优化建议

当前主包约 593 KB。优先保持 Command Center、认证和核心数据读取稳定，按访问频率拆分次级页面。

## 本阶段策略

- Command Center 保持同步加载，确保登录后首屏稳定。
- AI 成果、平台连接、系统状态、工作流与模型配置改为路由级动态加载。
- 不拆分内容工作台、Campaign 与策略、账号矩阵、素材库、角色库，避免影响高频生产流程。

## 后续建议

1. 将图片/视频预览和生成工作室提取成独立动态模块。
2. 对 300 条以上知识记录增加分页或虚拟列表，减少 DOM 数量。
3. 将图表或大型模型配置编辑器按需加载。
4. 使用构建产物分析工具定位最大依赖；不要仅提高 Vite warning 阈值来隐藏问题。
5. 每次拆包后验证 GitHub 登录恢复、Supabase 请求、页面导航和 GitHub Pages base path。
