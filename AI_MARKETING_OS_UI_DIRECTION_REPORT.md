# AI Marketing OS UI Direction Report

## 本次范围

本轮严格执行任务包的 Phase A，只继续完善“内容工作台”作为全站 AI Marketing Operations System 的交互样板。没有同时重构内容情报、账号矩阵、Command Center、素材库、角色库、发布队列或分析优化页面。

## 1. 本次改造了哪些页面

### 内容工作台

- 将内容生产主流程收敛为“文案确认 → 视觉生成 → 结果审核”。
- 保留左侧当前文案、右侧视觉生成和底部生成结果的工作台结构。
- 默认进入图片生成，也可切换视频生成。
- 角色下拉读取现有 `characters` 数据。
- LoRA 下拉读取当前角色及内容包已有的 LoRA 配置，支持多个历史兼容字段。
- 参考素材直接显示在视觉生成区，可通过缩略图选择或取消。
- 图片提示词 / 视频脚本继续关联当前文案、策略、角色、LoRA 和素材。
- 增加生成前检查，集中显示角色、LoRA、参考素材和执行服务状态。
- 保留完整设置、X 链接导入、本地上传、生产关联保存、结果回传和终审发布。

## 2. 哪些页面只做了结构调整

- 本轮没有修改其它页面。
- 内容工作台的 Supabase 数据读取、Context AI、执行网关、发布流程没有重写，只调整了界面结构和状态呈现。

## 3. 已经可以真实执行的按钮

- `AI 优化文案`
  - 调用现有 Context AI / Qwen Edge Function。
  - 生成结果需要人工点击“应用到当前内容”，不会自动覆盖。
- `保存草稿`
  - 通过现有安全执行网关提交。
- `保存当前关联`
  - 将策略、角色、LoRA、参考素材和生成模式保存回当前内容包。
- `立即生成图片`
  - 通过现有 `generate_character_image` 执行动作提交。
- `立即生成视频`
  - 通过现有 `generate_character_video` 执行动作提交。
- `上传到素材库`
  - 通过现有上传执行动作提交。
- `从 X 链接导入`
  - 通过现有导入执行动作提交。
- `审核通过并进入发布队列`
  - 继续使用现有终审与发布任务创建逻辑。

## 4. 仍可能等待 Bridge / API / 数据的按钮

- 图片或视频生成：
  - Bridge 未连接时阻塞，并显示执行服务原因。
  - 未选择角色时阻塞。
  - 角色没有 LoRA 时阻塞，并引导前往角色库。
  - LoRA 未启用对应图片或视频能力时阻塞。
  - 参考视频类生成缺少参考素材时阻塞。
- Context AI：
  - 千问 API Key、额度、模型授权或 Edge Function 不可用时会显示对应错误。
- 素材上传 / X 导入：
  - 缺少文件、有效 X 链接或素材使用权确认时阻塞。
- 终审发布：
  - 缺少正文、CTA 或可用素材时阻塞。
  - 真实发布仍受人工确认和发布保护约束。
